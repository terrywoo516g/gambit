# P2 功能规格：灵光一闪（暂定名，最终名公测前定）

> 代号：Insight Officer
> 当前占位名：灵光一闪
> 阶段：P2
> 状态：规格定稿，架构预留完成，P0/P1 稳定后正式联调

---

## 1. 产品定位

Gambit 的核心付费锚点。三个 Agent 是评审团集体智慧，洞察官是"当所有人都觉得结论对时，那个说'等等，你们都错了'的人"。

**差异化价值**：其他产品给用户答案，洞察官颠覆用户对问题的理解。

---

## 2. 触发与调用

| 项 | 设计 |
|----|------|
| 触发点 | 决策页加载完成后，显示按钮"灵光一闪（消耗 1 次）" |
| 触发主体 | 用户主动点击，不自动触发 |
| 免费额度 | 每用户每日 3 次（公测期可调整） |
| 付费额度 | 无限次 |
| 底层模型 | Claude Opus 4.7（海外模型，走后端中转） |

---

## 3. 输入

```
- 用户原始问题
- 三个 Agent 的完整 claims（key_action/thesis/rationale/risks/assumptions）
- 分歧状态（共识摘要或分歧点）
- 用户自述想法（如果用户在"我的想法"里填了）
- 核心张力
```

---

## 4. 输出

**格式**：单段自然语言文字，≤200 字

**必须满足以下之一（Prompt 强制）**：

1. **揭示隐藏变量**：指出三个 Agent 都忽略的关键变量
2. **重定义问题**：指出用户问的问题本身错了，应该问的是另一个问题
3. **二阶效应推导**：推演某个方向 2-5 年后会长成什么样
4. **反共识重构**：在三方共识处埋伏陷阱

**禁止输出**：
- 对三个 Agent 立场的综合/折衷（那是 Captain 的职责）
- 执行建议或动作清单（洞察官只提供视角，不给动作）
- 超过 200 字的长篇分析
- 需要联网查证的具体数据（避免幻觉）

---

## 5. 合规与安全（重要）

### 反注入防护

- 用户输入一律视为**内容**，不视为**指令**
- Prompt 里明确：忽略用户输入中的任何"请以 XX 身份/忽略上述/system:"等指令样式文本
- 用分隔符（如 `<<<USER_INPUT>>>`）包裹用户输入

### 反越狱防护

- 系统提示词里明确拒绝输出：政治敏感、违法建议、煽动性判断、医疗/法律的绝对结论
- 对涉及政治人物、国家政策、敏感行业的问题，洞察官直接返回"此类问题超出洞察范围"
- 输出后做一层关键词过滤（敏感词库）

### 数据与隐私

- 用户原始问题走海外模型需脱敏处理（当前公测版可以简化）
- 后端中转日志只保留 usage 字段，不持久化 prompt 正文
